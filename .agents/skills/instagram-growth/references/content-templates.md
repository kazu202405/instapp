## 1. フックテンプレート

### Type 1: 認知的不協和フック

常識と矛盾する主張で、解消欲求を生む。

**テンプレート:**
- 「○○をやめたら、△△が[良い結果]になった」
- 「[常識]は全部間違っていた」
- 「[成功したいなら][常識の逆]をしろ」
- 「[権威/実績]を捨てた理由」

**実例（ジャンル別）:**

| ジャンル | フック例 |
|---------|---------|
| ビジネス | 「毎日投稿をやめたら、フォロワーが3倍に増えた」 |
| 美容 | 「高い化粧品をやめたら、肌がきれいになった理由」 |
| 料理 | 「レシピ通りに作るのが一番まずい」 |
| 筋トレ | 「筋トレを週5から週2に減らしたら体が変わった」 |

### Type 2: 好奇心ギャップフック

知っていることと知りたいことの間にギャップを作る。

**テンプレート:**
- 「[X]%の人が知らない○○の事実」
- 「これを変えたら[指標]が[倍率]倍になった」
- 「プロが絶対に教えない[テーマ]の裏技」
- 「[テーマ]で一番大事なのに誰も言わないこと」

**実例:**

| ジャンル | フック例 |
|---------|---------|
| マーケ | 「エンゲージメントが3倍になった、たった1つの変更点」 |
| 投資 | 「資産1億円の人が絶対やらない3つのこと」 |
| 英語 | 「ネイティブが日本人に教えたくない英語の秘密」 |
| 子育て | 「小児科医が自分の子にだけやっていること」 |

### Type 3: 感情的衝撃フック

扁桃体を直接刺激する。

**テンプレート:**
- 「これ、今すぐやめてください」（恐怖/危険）
- 「信じられない結果が出た」（驚き）
- 「なぜ誰もこれを言わないのか」（怒り）
- 「[予想外のオチへの導入]」（笑い）

### Type 4: 自己関連性フック

ターゲットを名指しして注意を固定する。

**テンプレート:**
- 「○○な人、これ見て」
- 「[年代]で[状況]の人へ」
- 「[悩み]がある人の共通点」
- 「これ当てはまる人、ヤバいかも」

### Type 5: 数字・具体性フック

具体的数字で処理流暢性と信頼性を高める。

**テンプレート:**
- 「[期間]で[数字][成果]を達成した方法」
- 「たった[小さい数字]つの変更で[指標]が[大きい数字]倍に」
- 「[大きい数字]の人がやっている[テーマ]の習慣」

### Type 6: ストーリーフック

物語でオキシトシンを分泌させ、注意を持続させる。

**テンプレート:**
- 「[困難な状況]から始めた話を聞いてほしい」
- 「[X]年前の自分に伝えたいこと」
- 「この[出来事]が全てを変えた」
- 「あの日、[転機となる場面]」

## 3. CTAテンプレート

### 保存を促すCTA

心理原理: 損失回避 + 実装意図

**効果の高い順:**
1. 「このリストは何度も使えるから、保存して見返してね」（実用的価値の明示）
2. 「後で絶対必要になるから、今のうちに保存しておいて」（損失フレーミング）
3. 「右下のブックマークボタンをタップして保存📌」（具体的な行動指示）

**NG:** 「保存してね!」（理由なし = 動機不足）

### シェアを促すCTA

心理原理: 社会的通貨 + 利他性

**効果の高い順:**
1. 「○○に悩んでる友達に送ってあげて」（具体的な受取人の想起 + 利他性）
2. 「これ知らない人多いから、ストーリーズでシェアして広めて」（社会的使命感）
3. 「タメになったら友達にもシェアしてね」（返報性の軽い活用）

### コメントを促すCTA

心理原理: 自己表現欲求 + 選択の単純化

**効果の高い順:**
1. 「あなたはAタイプ？Bタイプ？コメントで教えて」（二択 = 認知負荷最小）
2. 「最も共感したポイントの番号をコメントして」（選択式 = 最小摩擦）
3. 「正直どう思う？コメントで聞かせて」（自己表現の場の提供）

**NG:** 「コメントしてね」（何を書くかわからない = 能力の壁）

### フォローを促すCTA

心理原理: 期待構築 + 損失回避

**効果の高い順:**
1. 「次回は○○について詳しく解説するから、見逃さないようにフォロー」（予告 + FOMO）
2. 「こういう情報を毎日発信してます◎」（一貫性の証明 = 信頼）
3. 「フォローして、一緒に○○を目指そう」（仲間意識の構築）

### CTA組み合わせの段階設計

フット・イン・ザ・ドア戦略（小→大）:
```
いいね（最小摩擦）
  → 保存（中程度）
    → コメント（高摩擦）
      → シェア（最高コミットメント）
        → フォロー（継続的コミットメント）
```

1投稿で狙うCTAは**最大2つ**（主CTA + 副CTA）。

## 4. カルーセル設計テンプレート

### 基本構造（10枚）

| スライド | 役割 | 心理メカニズム | 内容ガイド |
|---------|------|-------------|-----------|
| 1 | フック | パターンインタラプト | タイトル+「スワイプ→」 |
| 2 | 問題提起 | 共感（Neural Coupling） | 「こんな経験ない？」 |
| 3-8 | 価値提供 | 段階的開示、認知負荷分散 | 1スライド1ポイント |
| 9 | まとめ | チャンク化で記憶定着 | 全ポイントの一覧 |
| 10 | CTA | BJ Foggモデルのプロンプト | 保存+フォロー+シェア |

### 1枚目のデザイン原則

- テキストは**大きく、太く**（処理流暢性の最大化）
- 背景とのコントラストを強く
- 「スワイプ→」の視覚的キュー（ツァイガルニク効果の起動）
- 数字を入れる（「5つの方法」「3つの秘訣」）

### データ

- 最適スライド数: 7-10枚（情報密度と完了率のバランス）
- カルーセルの保存率: 単一画像の3-5倍
- 2回表示機能: 1回目でスワイプしなかった場合、2枚目を最初に表示して再表示される

### テーマ別テンプレート

**教育型:** 「○○の[数字]つの方法」
```
1: タイトル → 2: よくある悩み → 3-8: 方法1-6 → 9: まとめ → 10: CTA
```

**ビフォー/アフター型:**
```
1: 衝撃的な結果 → 2: ビフォーの状態 → 3-7: 変化のプロセス → 8-9: アフター → 10: CTA
```

**データ/統計型:**
```
1: 衝撃的な数字 → 2-8: データポイント+解説 → 9: 結論 → 10: CTA
```

## 5. リール構成テンプレート

### ドーパミンループ設計

```
[期待の構築 0-3秒] フック: 「この先に価値がある」予測
      ↓
[段階的開示 3-15秒] 情報ギャップを少しずつ埋める
      ↓
[クライマックス 最後5秒] 予測の確認 or 裏切り（予測誤差）
      ↓
[ループ誘発] 最後→最初につながる構造でリプレイ促進
```

### 長さ別の最適構成

| 長さ | 最適用途 | 構成 |
|------|---------|------|
| 7-15秒 | トレンド、ミーム、1ポイント | フック→オチ |
| 15-30秒 | ノウハウ、tips | フック→3ポイント→CTA |
| 30-60秒 | ストーリー、解説 | フック→問題→解決→CTA |
| 60-90秒 | 深い教育、体験談 | フック→ストーリー→価値→CTA |

### パフォーマンス最適化

| 要素 | 最適値 | 根拠 |
|------|--------|------|
| フック | 最初の1.5秒 | システム1の自動判断 |
| テキスト | 画面上部配置 | F型視覚走査パターン |
| 音楽 | トレンド音源 | 親近感ヒューリスティック |
| 字幕 | 必須 | 40%が無音視聴 |

## 6. ストーリーズ設計テンプレート

### 1日の基本フロー（3-7フレーム）

| フレーム | 目的 | 内容例 | インタラクション |
|---------|------|--------|---------------|
| 1 | 挨拶/日常 | おはよう、今日の予定 | なし（親近感） |
| 2 | 価値提供 | 今日のtip、気づき | アンケート or クイズ |
| 3 | 質問/対話 | フォロワーに質問 | 質問箱 or 絵文字スライダー |
| 4 | 裏側/プロセス | 作業中、制作過程 | なし（透明性） |
| 5 | CTA | フィード投稿への誘導 | リンク or 「最新投稿見てね」 |

### ストーリーズのピーク・エンドの法則

最後のフレームが全体の印象を決める。最終フレームには:
- 強いCTA
- 印象的なビジュアル
- 次回予告（ツァイガルニク効果）
