## 1. KPIの階層構造

### Tier 1: 成長指標（月次で確認）

| 指標 | 計算方法 | 意味 |
|------|---------|------|
| フォロワー増加率 | (今月末-先月末) / 先月末 × 100 | アカウント全体の成長速度 |
| フォロワー外リーチ率 | フォロワー外リーチ / 総リーチ × 100 | 新規への露出力 |
| プロフィール訪問率 | プロフィール訪問数 / 総リーチ × 100 | コンテンツ→プロフィールの転換力 |
| フォロー転換率 | 新規フォロワー / プロフィール訪問数 × 100 | プロフィールの説得力 |

### Tier 2: エンゲージメント指標（週次で確認）

| 指標 | 計算方法 | 重要度 | 意味 |
|------|---------|--------|------|
| 保存率 | 保存数 / リーチ数 × 100 | 最重要 | コンテンツの参照価値 |
| シェア率 | シェア数 / リーチ数 × 100 | 2番目 | コンテンツの共有価値（2025年最重要シグナル） |
| コメント率 | コメント数 / リーチ数 × 100 | 3番目 | コンテンツの対話喚起力 |
| エンゲージメント率 | (いいね+コメント+保存+シェア) / リーチ × 100 | 総合 | 全体のエンゲージメント |

### Tier 3: コンテンツ品質指標（投稿別で確認）

| 指標 | 対象 | 意味 |
|------|------|------|
| 視聴完了率 | リール | 最後まで見た割合 |
| リプレイ率 | リール | 繰り返し再生された割合 |
| スワイプ率 | カルーセル | 何枚目まで見られたか |
| 滞在時間 | 全フォーマット | どれだけ長く投稿に留まったか |
| 「もっと見る」タップ率 | 全フォーマット | キャプションの続きを読んだ割合 |

|---------------|------|------|-----------|
| 1K未満 | 5%+ | 8%+ | 12%+ |

## 3. 投稿分析フレームワーク

### ステップ1: 数値の確認

以下のデータを収集:
- リーチ（フォロワー内 / フォロワー外の比率）
- いいね数、コメント数、保存数、シェア数
- プロフィール訪問数、フォロー数
- （リールの場合）視聴完了率、リプレイ率
- （カルーセルの場合）スワイプ率

### ステップ2: ベンチマークとの比較

3つの基準で比較:
1. **業界平均**（上記ベンチマーク表）
2. **自分の過去平均**（直近30日）
3. **自分のベスト投稿**

### ステップ3: 行動科学的分析

「なぜこの結果になったか」を以下の観点で分析:

| 分析ポイント | 確認事項 |
|-------------|---------|
| フック | 1行目は効果的だったか？どの類型を使ったか？ |
| 感情設計 | どの感情を狙ったか？実際に反応を引き出せたか？ |
| 認知負荷 | 情報量は適切だったか？読みやすかったか？ |
| CTA | 具体的だったか？理由はあったか？ |
| タイミング | フォロワーのアクティブ時間に投稿したか？ |
| フォーマット | 目的に最適なフォーマットだったか？ |
| ハッシュタグ/SEO | 適切なキーワードが含まれていたか？ |

### ステップ4: 改善アクションの策定

分析結果から、次回の投稿で試すべき具体的なアクションを3つ提案。
各アクションには行動科学的な根拠を付ける。

## 4. A/Bテスト設計

### 仮説テンプレート

```
仮説: [心理原理]に基づき、
      [変更A]は[変更B]より
      [指標]が[予測値]%向上する

根拠: [なぜそう予測するかの行動科学的説明]
```

### テスト可能な変数

| 変数 | A案 | B案 | 測定指標 | 心理原理 |
|------|-----|-----|---------|---------|
| フック類型 | 質問形式 | 衝撃的事実 | リーチ、完了率 | 好奇心ギャップ vs 感情的衝撃 |
| CTAの具体性 | 「保存して」 | 「保存して後で見返して」 | 保存率 | BJ Foggの能力+動機 |
| フレーミング | 利得フレーム | 損失フレーム | エンゲージメント率 | プロスペクト理論 |
| 色使い | 暖色系 | 寒色系 | スクロール停止率 | 色彩心理学 |
| 顔出し | あり | なし | フォロー率 | 紡錘状顔領域の活性化 |
| リール長さ | 15秒 | 60秒 | 完了率 vs リーチ | 認知負荷理論 |
| 数字の提示 | あり（具体的数字） | なし（定性的） | 保存率 | 処理流暢性 |
| CTA位置 | キャプション冒頭 | キャプション末尾 | 対象アクション率 | 初頭効果 vs 新近効果 |
| 投稿時間 | 朝（7-9時） | 夜（19-21時） | 初速エンゲージメント | 時間帯別認知状態 |

### テストの実施ルール

1. **1回のテストで変更は1変数だけ**（他の要素は統一）
2. **最低5投稿ずつ**でテスト（統計的に有意な差を確認）
3. **同じ曜日・時間帯**で比較（タイミングの影響排除）
4. **結果の記録**:

```
テスト#: [番号]
変数: [テスト対象]
A: [条件A]
B: [条件B]
期間: [日付範囲]
投稿数: A=[数] / B=[数]
結果:
  - [指標1]: A=[値] / B=[値]（差=[%]）
  - [指標2]: A=[値] / B=[値]（差=[%]）
勝者: [A or B]
学び: [何がわかったか]
次のアクション: [結果を踏まえた次の施策]
```

## 5. 週次・月次レビューガイド

### 週次レビュー（15分）

1. **今週の投稿パフォーマンス一覧**
   - 各投稿のリーチ、保存率、シェア率
   - ベスト投稿とワースト投稿の特定

2. **ベスト投稿の分析**
   - なぜ伸びたか？（フック？フォーマット？タイミング？）
   - 再現可能な要素は何か？

3. **ワースト投稿の分析**
   - なぜ伸びなかったか？
   - 避けるべきパターンは何か？

4. **来週の改善ポイント**
   - 今週の学びを来週のカレンダーに反映

### 月次レビュー（30分）

1. **成長指標の確認**
   - フォロワー増加率
   - 月間リーチの推移
   - プロフィール訪問→フォロー転換率

2. **コンテンツピラーのバランス確認**
   - 教育 40% / インスピレーション 30% / つながり 30%
   - バランスが崩れていないか？

3. **フォーマット別パフォーマンス**
   - リール vs カルーセル vs 単一画像
   - どのフォーマットが最も効果的か？

4. **A/Bテスト結果のまとめ**
   - 今月のテスト結果
   - 来月のテスト計画

5. **トップ5投稿の共通パターン**
   - フック類型、感情、フォーマット、投稿時間
   - 「勝ちパターン」の特定

6. **来月の戦略調整**
   - コンテンツカレンダーの方針修正
   - 新しいテスト仮説の策定

## 6. 成長ステージ別の重点KPI

### Stage 1: 0-1,000フォロワー（立ち上げ期）

**最優先KPI**: フォロワー外リーチ率、フォロー転換率

| 重点指標 | 目標 | 理由 |
|---------|------|------|
| 週間リール投稿数 | 3本以上 | 非フォロワーへのリーチが最大のフォーマット |
| リール視聴完了率 | 40%+ | アルゴリズムに評価されるための最低ライン |
| プロフィール訪問→フォロー率 | 15%+ | プロフィールの説得力の指標 |

**この時期の戦略**: リールで広くリーチし、プロフィールで確実に転換する。
コンテンツの質よりも「量と一貫性」を重視。

### Stage 2: 1,000-10,000フォロワー（成長期）

**最優先KPI**: 保存率、シェア率

| 重点指標 | 目標 | 理由 |
|---------|------|------|
| 保存率 | 3%+ | コンテンツの質がアルゴリズムに評価される |
| シェア率 | 1.5%+ | 2025年最重要の成長シグナル |
| エンゲージメント率 | 4%+ | フォロワーとの関係性の深さ |

**この時期の戦略**: カルーセルで保存率を稼ぎ、リールでシェアを増やす。
コンテンツの質を上げ、「保存・シェアされるコンテンツ」を量産する。

### Stage 3: 10,000-100,000フォロワー（拡大期）

**最優先KPI**: シェア率、コミュニティエンゲージメント

| 重点指標 | 目標 | 理由 |
|---------|------|------|
| シェア率 | 2%+ | 非フォロワーへの有機的な拡散 |
| ストーリーズ閲覧率 | 10%+ | コアフォロワーの維持 |
| DM数 | 増加トレンド | コミュニティの深さ |

**この時期の戦略**: コミュニティ構築を本格化。ストーリーズとDMで関係を深め、
コアファンが自発的にシェアしてくれるエコシステムを作る。

### Stage 4: 100,000フォロワー以上（確立期）

**最優先KPI**: ストーリーズ閲覧率、フォロワーの質

| 重点指標 | 目標 | 理由 |
|---------|------|------|
| ストーリーズ閲覧率 | 8%+ | エンゲージメントの健全性 |
| シェア率 | 維持 | 成長の持続性 |
| UGC投稿数 | 増加トレンド | コミュニティの自走力 |

**この時期の戦略**: フォロワー数よりエンゲージメントの質を重視。
ファンが「布教者」になる仕組みを整える。

## 補足: インサイトの読み方

### Instagram Insightsで確認できる主要データ

| データ | 場所 | 確認頻度 |
|--------|------|---------|
| リーチしたアカウント | インサイト→概要 | 週次 |
| エンゲージしたアカウント | インサイト→概要 | 週次 |
| コンテンツ別パフォーマンス | インサイト→共有コンテンツ | 投稿毎 |
| フォロワー属性（年齢、地域、性別） | インサイト→フォロワー | 月次 |
| 最もアクティブな時間 | インサイト→フォロワー | 月次 |

### 投稿個別のインサイト

各投稿をタップ→「インサイトを見る」で確認:
- リーチ（フォロワー / 非フォロワーの内訳）
- インプレッション（ホーム / Explore / ハッシュタグ / プロフィールの内訳）
- エンゲージメント（いいね / コメント / 保存 / シェアの内訳）
- プロフィールへのアクセス数
- フォロー数（この投稿からのフォロー）
