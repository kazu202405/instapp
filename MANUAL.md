# InstaGrowth 説明書

行動科学ベースのInstagram成長支援ツール。
外部API不要・費用ゼロで、キャプション生成から投稿分析まで一括管理。

---

## セットアップ

### 必要なもの

- Node.js v18以上（https://nodejs.org/）
- npm（Node.jsに同梱）

### 起動手順

```bash
# 1. プロジェクトフォルダに移動
cd Desktop/system/claude/instapp

# 2. パッケージインストール（初回のみ）
npm install

# 3. 開発サーバー起動
npm run dev
```

ブラウザで http://localhost:3000 を開く。

### よくあるエラー

**「Port 3000 is in use」と表示される場合**
→ 既にサーバーが動いています。http://localhost:3000 をそのまま開けばOK。

**「Unable to acquire lock」と表示される場合**
→ 別のdev serverプロセスが動いています。以下のどちらかで解決：
- そのまま http://localhost:3000 を開く
- またはターミナルで `taskkill /F /PID <プロセス番号>` で既存プロセスを終了してから再起動

## 機能一覧

### 1. ダッシュボード（ホーム）

アプリ全体の統計を一覧表示するトップページ。

- 生成キャプション数、平均スコア、プロフィールスコア、進行中テスト数
- パフォーマンス推移グラフ（投稿分析データがある場合）
- 最近のアクティビティ
- フォロワー成長シミュレーター
- 初回はオンボーディングカードが表示される

### 2. キャプション生成

行動科学ベースの4幕構造キャプションを自動生成。

**使い方：**
1. ジャンルを選択（ビジネス、フィットネス、料理など10種類）
2. テーマを入力（例：「朝のルーティン」「ダイエット成功の秘訣」）
3. キーワードを入力（カンマ区切りで複数可、任意）
4. 狙うアクション（保存、シェア、コメント、フォロー）を選択
5. フックタイプ（好奇心ギャップ、議論・逆張りなど6種類）を選択
6. 「生成する」をクリック → 3つのバリエーションが表示される

**生成後にできること：**
- 「この案を採用」→ 単一表示に切替
- 「コピー」→ クリップボードにコピー（Instagramに貼り付け用）
- 「カレンダーに予約」→ 投稿予定日をカレンダーに登録
- 「なぜこのフック？」→ 行動科学的な選択理由を表示

**履歴機能：**
- 生成したキャプションは自動保存される
- 星アイコンでお気に入り登録 → フィルターで絞り込み可能
- CSVエクスポートで一括ダウンロード
- 削除時は「元に戻す」ボタン付きトーストが表示される

### 3. テンプレート

ジャンル別のキャプションテンプレート集。

**使い方：**
1. ジャンルタブを選択
2. テンプレートカードをクリック
3. 「このテンプレートを使う」→ キャプション生成ページに遷移して自動入力

### 4. リール台本

Instagram Reelsの台本を自動生成。

**使い方：**
1. ジャンル・テーマ・尺（15秒/30秒/60秒/90秒）を設定
2. 「台本を生成」をクリック
3. フック→本編→CTA の構成で台本が生成される
4. 各セクションにタイムスタンプと演出ノート付き

### 5. コンテンツカレンダー

投稿計画を自動生成・管理。

**使い方：**
1. ジャンルを選択し、週の投稿数をスライダーで設定（1〜7回）
2. 「週間プラン生成」または「月間プラン生成」をクリック
3. 投稿予定がカードで表示される

**カレンダーの機能：**
- 週間表示 / 月間表示の切替
- カードをドラッグ&ドロップで日付移動（モバイルは長押し）
- 各カードの「キャプション生成」リンクから直接キャプション生成へ遷移（テーマ・フックが自動入力）
- コンテンツピラー配分（教育40%/インスピレーション30%/つながり30%）

### 6. 投稿分析

投稿パフォーマンスをスコア化・診断。

**使い方：**
1. Instagramのインサイト画面から数値を確認
2. 投稿タイプ、フォロワー数、リーチ、いいね、コメント、保存数、シェア数を入力
3. 「分析する」をクリック
4. 100点満点のスコア + ベンチマーク比較 + 改善アドバイスが表示

**結果の見方：**
- 総合スコア：100点満点
- エンゲージメント率、保存率、シェア率を個別評価
- フォロワー規模別ベンチマークとの比較グラフ
- 行動科学に基づく改善提案3つ

### 7. プロフィール最適化

心理学ベースのチェックリストでプロフィールを改善。

**項目：**
- ユーザーネーム（SEO最適化）
- プロフィール写真
- バイオ（自己紹介文）
- CTA（行動喚起）
- ハイライト
- 固定投稿

各項目にチェックを入れると総合スコアが更新される。
各項目には「なぜ重要か」の行動科学的理由が表示される。

### 8. A/Bテスト

投稿の仮説検証を記録・管理。

**使い方：**
1. テスト名・仮説を入力
2. バリエーションA/Bの内容を記入
3. テストを開始 → 結果を記録 → 学びをメモ
4. 心理原理ベースの仮説テンプレートから選択可能

### 9. 設定

アプリ全体の設定を管理。

- テーマ切替（ダーク/ライト/システム）
- デフォルトジャンル・投稿時間の設定
- データ管理（使用量確認、全データリセット）
- アプリ情報

## データについて

### 保存場所

全てのデータはブラウザの**localStorage**に保存される。
サーバーやクラウドには一切送信されない。

### データのバックアップ

サイドバー下部の「データ管理」ボタンから：

- **全データバックアップ（JSON）**: 全機能のデータを1ファイルでエクスポート
- **キャプション履歴（CSV）**: キャプション履歴をExcel対応CSVでエクスポート
- **カレンダー（CSV）**: カレンダー予定をExcel対応CSVでエクスポート
- **インポート**: バックアップしたJSONファイルから復元

### データが消えるケース

以下の場合、データが消える可能性がある：

- ブラウザのキャッシュ/データを削除した場合
- シークレットモード（プライベートブラウズ）で使用した場合
- 別のブラウザ・別のPCで開いた場合（データは端末ごと）

定期的にJSONバックアップを取ることを推奨。

## 使い方の流れ（推奨ワークフロー）

```
1. カレンダーで週間/月間の投稿計画を立てる
     ↓
2. カレンダーから「キャプション生成」をクリック
     ↓
3. キャプションを生成 → お気に入りに登録 → コピー
     ↓
4. Instagramアプリを開いて投稿（キャプションを貼り付け）
     ↓
5. 数日後、Instagramインサイトの数値を「投稿分析」に入力
     ↓
6. 分析結果をもとに次の投稿を改善
     ↓
7. A/Bテストで仮説を立てて検証
```

## デプロイ（公開する場合）

### Vercel（推奨・無料）

```bash
# ビルド確認
npm run build

# Vercelにデプロイ
npx vercel
```

Vercelアカウント（無料）が必要。
デプロイ後はURLが発行され、スマホからもアクセス可能。

### その他

- Netlify、Cloudflare Pages等でも静的サイトとしてデプロイ可能
- 環境変数やデータベースの設定は一切不要
